## 依頼内容（毎回このまま貼る）
あなたには「コードを丸投げで書いてもらう」のではなく、私が書けるように教えつつ、難しいところ・新しく学ぶ部分を詳細に解説してほしいです。

### 教え方の希望（重要）
- まず「次に実装する機能/デザイン」と全体設計（データ構造・画面構造・イベントの流れ）を提示
- 次に“完成形の全体コード”をあなたの回答に提示（ただし私は写経ではなく理解しながら自分で打ちたい）
- 最後に、そのコードの中で私が新しく学ぶべき点を重点的に詳説（なぜそう書くか、代案、注意点、よくあるバグ）
- 私が手を動かせるように、課題を小さく分けて「次にやること」を具体的に指示してほしい

### 用語の扱い
- 専門用語はその都度、短く定義してから使ってほしい（例: state / source of truth / render / イベント委譲）
- 説明するときは「該当コード（短い抜粋）」も一緒に表示してほしい

### 参照してほしいもの
- このワークスペース（プロジェクト）の現在のコードを確認して、進捗評価（できている点/改善点）と次の指示を出してほしい
- 進捗メモは `PROGRESS.md` を読むこと
- Todoアプリの作業場所は `Todo/vanilla-js-todo/`（`index.html` と `js/`）
